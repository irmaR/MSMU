Problem #1: Rock Paper Scissors

In this problem you will implement a few versions of the classic game Rock Paper Scissors, with a person playing against the computer. Each version will be implemented as a Python function. A player's choice is represented as a string and can be assumed to always be one of the three strings "rock", "paper", and "scissors". The output of each function is a string representing the result of the game, which must be one of the three strings "person wins", "computer wins", and "tie game". The file rps.py contains declarations of these functions but with incorrect implementations; your job is to update them. There are four functions for you to implement:
  1. Implement a function rpsCheat that takes the person's choice as an argument and cheats so that the computer always wins. Hint: the body of this function is just one line of code!

  2. Implement a function rpsRock that takes the person's choice as an argument and uses a strategy whereby the computer always selects "rock" as its choice.

  3. Implement a function rps2 that takes both the person's and the computer's choices as arguments (in that order). Note that and is an operator in Python that performs the logical "and" operation on booleans (i.e., values True and False). For example, the expression x>0 and y>0 evaluates to True if both x and y are positive and otherwise evaluates to False. The and operator will allow you to test the values of both players' choices in one expression. Hint: Since there are 9 possible combinations of values for the person's and computer's choice, you could use an if statement with 9 different branches. However, it's actually possible to implement this function in a simpler way that avoids testing all 9 combinations, instead requiring only 5 branches. How?

  4. Implement a function rpsRandom that takes the person's choice as an argument and picks the computer's choice randomly. The easiest way to make this random choice is to use the choice function from the random library, which I've imported for you (see the top of the file). Specifically, the expression choice(["rock", "paper", "scissors"]) passes to the function choice a list of the three possible options for the computer (we will study lists in Week 2 of the quarter), and choice randomly returns one member of this list. Rather than reimplementing the logic of the game again, this function should call your earlier function rps2 to do most of the work.
